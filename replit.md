# FS Exam Study Guide

## Overview
This project is an interactive study guide for the Fundamentals of Surveying (FS) exam, designed to help surveying students prepare for the NCEES FS exam. It offers structured study plans, practice quizzes, flashcards, and full-length practice exams, covering all 7 NCEES domains. The application aims to provide a comprehensive and engaging learning experience, guiding users toward becoming a professional licensed surveyor (PLS). The project also supports the Professional Surveying (PS) exam.

## System Architecture
The application features a comprehensive UI/UX with domain-specific color coding, Inter font for UI, and JetBrains Mono for code/formulas, designed responsively with dark mode support.

**Key Features:**
-   **Study Plans:** 16-week structured plans with flexible modes (Standard, Result-Driven, Working Professional, Custom), progress tracking, and textbook chapter recommendations.
-   **Practice & Assessment:** Mixed Exam and Domain Practice Quizzes with instant feedback and 4-Step Problem-Solving Loop (Understand → Formula → Units → Sense-Check); Enhanced Flashcard System (Quick Review, Challenge Mode, Triad Drill, Feynman Mode, Mnemonic Builder); Full Practice Exam Simulator (110 questions) and NCEES-Style Practice Exam (60 questions, including AITs).
-   **Personalized Learning:** Custom Plan Builder; Study Cycles System for multi-pass learning; Adaptive Difficulty System based on user performance; Optimal Review Timing using SM-2 algorithm for spaced repetition.
-   **Engagement & Motivation:** Progress Dashboard with Gamification (XP System with 10 Surveyor Ranks, Study Streaks, Achievements); Daily Quests System; Weekly Leaderboard.
-   **AI-Powered Guidance:** AI Study Coach Briefing provides personalized daily guidance, focus recommendations, and progress insights.
-   **Study Tools:** Enhanced Study Notes with CRUD operations; Study Resources library; Interactive Lessons System (Duolingo-style, domain-based); Procedures & Standards reference with BLM Glossary and exam traps.
-   **Analytics & Insights:** Personal Analytics Dashboard with learning velocity, time investment ROI, weakness predictions, and progress trajectory; Forgetting Curve Visualization.
-   **Content & Coverage:** Expanded Question Pool (2,290 questions from 458 archetypes) with variation system; Complete NCEES coverage for all 7 FS domains and 5 PS domains; Dual textbook reference (SRM and Elementary Surveying). Interactive Study Readings (34 FS readings across all domains with ~350+ sections of original content). Expanded flashcard deck (114 FS cards + PS cards). Additional archetypes sourced from "1001 Solved Surveying Fundamentals Problems" covering 13 topic areas stored in `shared/questionArchetypesExpanded.ts`.
-   **Interactive Study Readings:** 34 readings covering all 7 FS domains plus exam standards walkthroughs, with structured content blocks (concept, formula, worked_example, knowledge_check, further_reading). Integrated into Study Plan's READ section via week-to-reading mapping (weeks 1-15). Progress tracked per section with 5 XP per completed section (first completion only). Content files: `shared/data/studyReadings.ts` (D0-D1), `studyReadingsD2.ts`, `studyReadingsD3.ts`, `studyReadingsD4D6.ts`, `studyReadingsD5.ts`, `studyReadingsD5Expanded.ts` (boundary law), `studyReadingsD4Expanded.ts` (geodesy), `studyReadingsD6Expanded.ts` (professional practice), `studyReadingsD7Expanded.ts` (applied math/stats), `studyReadingsD1D2Expanded.ts` (field methods/mapping), `studyReadingsStandards.ts` (ALTA, FEMA, FGCS, NSPS walkthroughs).
-   **Multi-Exam Support:** Supports both FS (73 lessons, 8 domains) and PS (55 lessons, 5 domains) exams with isolated experiences.

**Technical Implementation:**
-   **Frontend:** React with TypeScript, Tailwind CSS, Shadcn UI, Wouter, TanStack Query, and Lucide React.
-   **Backend:** Express.js server.
-   **Database:** PostgreSQL with Drizzle ORM.
-   **Authentication:** Replit Auth.
-   **Validation:** Zod.

**System Design:**
-   Separate question pools for quizzes and exams with stable IDs.
-   User-scoped data storage with protected API routes.
-   Study Cycles Architecture for tracking progress within cycles.
-   Server-side validation and security for interactive lessons.
-   Domain-Based Lesson Architecture for flexible study distribution.
-   Question Archetype System for deterministic question generation and variations using seeded RNG.
-   XP Idempotency System to prevent duplicate XP awards using activity keys.
-   Reading Checkpoint Completion Guard: Uses immutable `completedAt` timestamp to prevent XP farming via toggle off/on cycles. XP and quest progress only awarded on first-ever completion.
-   Weak Domain Quest Guard: Requires minimum 50% accuracy AND session-based idempotency to prevent farming via repeated low-effort quizzes.
-   Multi-Exam Architecture with isolated routes (`/app/:examTrack/*`) for FS and PS exams, allowing for distinct content and future monetization.
-   ExamTrackProvider Context for centralized exam track state and configuration.
-   Flashcard Session Resume System: Sessions persist user state (deck, domains, card index, study mode) and can be resumed from exact position. Review events logged individually for accurate statistics and streak tracking.
-   Flashcard Challenge Mode: Term-matching game mode where definitions are shown and users select the correct term from a bank. Multi-round progression: all cards in domain batched in rounds of 8, with inter-round summary screens and cumulative stats tracking. Tracks first-try accuracy, wrong attempts, and per-domain proficiency. Awards XP on completion (10-25 based on accuracy). Session results persisted to `flashcard_challenge_sessions` table. Challenge proficiency tab in Progress page shows sessions played, cards attempted, overall accuracy, and per-domain breakdown. Challenge Mode Session Persistence: Progress saved to localStorage on every card match and round completion. Persists card ordering, cumulative stats, and in-round state (matchedIds, attempts, accuracy). Resume prompt shown on return with progress summary. 24-hour expiry. Card order validated on resume (80% match threshold). API routes: POST `/api/flashcards/challenge/complete`, GET `/api/flashcards/challenge/sessions`, GET `/api/flashcards/challenge/stats`. Component: `client/src/components/flashcard-challenge-mode.tsx`.
-   PS Exam Track Content Scoping: Sidebar navigation hides FS-only items (Resources) for PS track via `tracks` property on menu items. Dashboard and Progress page conditionally hide FS-only resource links. PS has its own interactive study readings (20 readings across 5 domains in `shared/data/studyReadingsPS.ts`), quiz questions (248 questions in `shared/data/psQuizQuestions.ts`), and study plan (12-week structure via PS_STUDY_PLAN). PS_WEEK_TO_READING_IDS mapping integrates PS readings into READ section. Procedures, lessons, quizzes, flashcards, and practice exams properly filter by examTrack.
-   Exam Track Data Isolation: `weekProgress`, `quizSessions`, `readingProgress`, `applyChallengeAttempts`, and `retentionReviews` tables have `examTrack` column (default 'fs') for proper data separation. All progress APIs filter by examTrack: `/api/progress/weeks`, `/api/progress/weeks/:week`, `/api/progress/overall`, `/api/progress/stats`, `/api/quiz/sessions`, `/api/quiz/stats`, `/api/exams`, `/api/reading-progress`, `/api/apply/attempts`, `/api/retention/*`, `/api/focus/*`. Practice exams filtered by checking domainScores keys against FS_DOMAINS/PS_DOMAINS. Focus/weakness scanner filters quiz results by domain name arrays (FS vs PS domains). Client passes examTrack in all progress query keys and API calls via array-based query keys. Cache invalidation uses examTrack-scoped query keys throughout all study plan section components (ReadCheckpoint, FocusWeaknessScanner, ApplyScenarioLab, ReinforceRetentionBooster). Foundation for future Stripe-based guide monetization with isolated progress tracking per exam.
-   Retention Booster User Ownership: Client-side validation ensures reviews belong to current user before use. Cache control headers prevent CDN/browser caching of user-specific API responses. Auto-recovery creates fresh reviews when foreign data detected or 403 errors occur.
-   Timezone-Aware Quest System: Daily quests reset at the user's local midnight, not a rolling 24-hour window. User timezone is auto-detected from browser (`Intl.DateTimeFormat`) and stored in preferences. Server-side `getLocalMidnight(timezone)` helper calculates UTC boundaries for the user's local day. All quest generation, retrieval, and progress tracking use timezone-aware date comparisons.
-   4-Step Problem-Solving Loop: Integrated into practice quizzes as optional guidance (not a prerequisite). Expandable "Stuck? Try the 4-step breakdown" link appears above the Submit button. Steps: (1) What is the problem asking? (2) Which formula/concept applies? (3) Check units. (4) Does the answer make sense? Each step has guided prompts and optional notes. Component resets per question via React key. Component: `client/src/components/problem-solving-loop.tsx`.
-   Comprehensive Domain Mastery: `getDomainMastery` in `server/storage.ts` aggregates 8 data sources per domain: lessons, quiz results, flashcard quick review mastery (via `comp-card-{index}`/`card-{index}` ID mapping to FLASHCARDS domain field), challenge mode accuracy (domainBreakdown), practice exam domain scores, retention booster mastery levels, reading progress (partial completion via `{examTrack}-d{N}-` prefix matching), and apply scenario completions (via week-to-domain study plan mapping). Weighted composite scoring: Quiz 30%, Lessons 20%, Flashcards 15%, Exams 15%, Challenge 10%, Retention 10%, Reading 5%, Apply 5%. Only active sources contribute weights. Progress page shows per-source scores and source badges.
-   Daily Discipline Quest Auto-Progression: Natural study activities automatically map to pillar quest progress: Quiz completion=FOCUS, Lesson completion=REINFORCE, Apply scenario=APPLY, Reading activity=READ. Uses xpGrants table with activityKey pattern `pillar_quest:{examTrack}:{date}:{pillar}` for per-day idempotency (one credit per pillar per day). Method: `updatePillarQuestProgress` in `server/storage.ts`.
-   Study Reading System: Interactive readings with structured content blocks. Progress tracked per section in `study_reading_progress` table. 5 XP per section (first completion only via activityKey `reading:{readingId}:{sectionId}`). Week-to-reading mapping in `WEEK_TO_READING_IDS` (read-checkpoint.tsx) links study plan READ section to interactive readings. Readings accessible from sidebar and from within study plan weeks.
